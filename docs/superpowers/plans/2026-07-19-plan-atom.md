# Plan Atom Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add atom `plan` with linked layer group, right-side Track texture palette (filter + height → auto width), and DnD of `image` tiles onto the canvas.

**Architecture:** Creating a Plan creates a group + child `plan` element (`tileGroupId`). `PlanTilesPanel` shows when the plan or a linked tile is selected. Drops create snapped `image` children in that group. Reuse `useTrackTextures`, `nudgeItemInStack`, and existing `image` / `rotation` behavior.

**Tech Stack:** Vue 3 Composition API, Pinia, existing editor mm/snap helpers, `node --test`.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-19-plan-atom-design.md`
- Depends on Track textures API / `useTrackTextures` (already on this branch)
- Tiles = `image` atoms with `mediaId` + `textureId`
- Panel default size: **height mm only**; width = height × (width_px/height_px); fallback square if no px
- After drop, tile is autonomous (resize/rotate free)
- Plan marker stays first child; Cmd+[ ] nudges among `image` siblings only
- Delete plan atom → delete entire linked group
- Guide visible in editor only; hidden in print/preview
- Update `specs/WORKPLAN.md` at end
